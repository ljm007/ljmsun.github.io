#####1. git 回退
git回退可以用reset命令来实现，git reset –hard id
回退前先用git log查看提交历史，看要回退到哪个版本，更进一步还可以用git reflog
reset一般有三种使用方式：

  • git reset –mixed id 这是默认的方式，直接写 git reset id ，就是这个方式，这只会回退commit和index，所有的修改还都在，源码都没有发生变化，变的都只是commit信息回退到某个版本
  • git reset –soft id 这只回退了commit，index即暂存区的信息还在，id前面的提交都保存到暂存区里了。这种应用场景一般都是commit信息写的不对想回退重新写commit
  • git reset –hard id 完完全全的回退到某个版本，源码会跟回退的版本一模一样

一般用git log 可以查看提交历史

commit 后面的一大串哈希值就是提交的id，回退时一般只需要输入前五位的值就可以，就像现在我要回退到‘555’的版本，输入 git reset 18e01即可
用git reflog 可以查看所有的历史操作

#####2. git 提交代码冲突
有时候我们在写完若干个提交后push到主分支发生冲突，主要是我们没有在push之前先pull远程仓库的更新到本地分支
